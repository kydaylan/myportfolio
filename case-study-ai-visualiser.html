The Brief

eventhireGroup needed a way to show event planners photorealistic table settings for products that may not be physically available or set up in venue contexts. This was needed as part of a "Premier Partners" retention service for high-value B2B account customers.

The Challenge

Creating photorealistic product mockups for event hire is significantly harder than it sounds. AI image generation can produce impressive results, but getting it to accurately replicate specific real products — and arrange multiple products together in a believable venue setting — requires far more than a text prompt.

Product Accuracy

AI needs to reproduce exact replicas of real inventory — specific chair designs, tableware, linen colours. Generic "a table setting" is useless. Every product file has to be named and referenced individually in the prompt.

Multi-Product Composition

A single table setting might include 6–8 different products. Getting AI to place multiple specific items in one image with correct scale, spacing, and perspective is the hardest part of the process.

Venue Context

Products need to sit convincingly in real client venues. Sometimes that means clearing existing furniture from a venue photo with AI before placing new products in.

Every Image Is Different

No two client requests are the same — different products, different venues, different layouts. The workflow has to adapt to each brief rather than following a fixed template.

What I Built

I built a web application using Claude Code and Google AI Studio (Gemini) that automates the standard workflow — uploading products, selecting venues, and generating mockups. It works well for routine requests and is simple enough for colleagues to use without me.

  • Upload product photos directly into the application
  • Configure table layouts including number of seats and arrangement style
  • Select from pre-loaded marquee/venue backgrounds or upload custom backgrounds
  • JSON-based image generation pipeline for consistent, repeatable outputs

But in practice, the best results come from bespoke work. Each image is different — different products, different venues, different challenges — and the approach has to adapt every time. The app handles the straightforward requests; the complex ones need hands-on prompt engineering and often post-processing in Photoshop.

The Real Workflow

The skill behind these images isn't pressing a button. Every output requires a different approach depending on the products, the venue, and the brief. My workflow has evolved significantly over time:

Prompt Engineering

Each product file is individually named and referenced in the prompt with specific placement instructions — seat count, arrangement, which product goes where. A single image might need a prompt referencing 8+ individual files with precise layout directions.

Evolved from Text to JSON

Started with standard text prompts written in ChatGPT for Google AI Studio. Over time, moved to structured JSON-specific prompts for more reliable, repeatable results with complex multi-product compositions.

Venue Preparation

Client venue photos often have existing furniture that needs removing first. AI is used to clear the room before placing new products in — a multi-step process where each stage builds on the last.

Photoshop Compositing

When AI alone can't achieve the right result, the workflow combines AI-generated elements with Photoshop compositing — placing AI products into real venue backdrops, adjusting lighting and perspective manually.

The images that look effortless are the ones that took the most iteration. AI is a powerful tool, but the quality comes from knowing how to direct it — and knowing when to step in manually.

How It Evolved

This wasn't a single build — it's a workflow that has evolved over months of real client work, with each brief pushing the approach further.

Text prompts in ChatGPT + Google AI Studio

Initial approach — writing descriptive text prompts and feeding them into Gemini. Worked for simple single-product images but broke down with complex compositions.

Structured JSON prompts

Moved to JSON-formatted prompts that reference individual product files by name with specific placement instructions. Far more reliable for multi-product scenes.

Multi-step generation

Developed techniques for clearing existing furniture from venue photos before placing new products — turning a single generation into a multi-step pipeline.

Photoshop compositing layer

Added manual compositing for images where AI alone couldn't achieve the right result — combining AI-generated elements with real venue photography.

Web app for standard requests

Built a web application using Claude Code to automate the standard workflow — product upload, venue selection, and generation — so colleagues can handle routine requests independently.

Team training and handover

Trained colleagues on both the app and the manual workflow, so the service doesn't depend on one person.

What I Learned

AI image generation is powerful but it's not magic. The gap between "a nice image" and "an accurate representation of our actual products in a real client venue" is enormous, and closing that gap is where the real skill sits.

The workflow changes from image to image. Some work first time with the right prompt. Others need the venue clearing first, then furniture placement, then Photoshop correction. Knowing which approach to take — and when to abandon one method for another — is something that only comes from doing it repeatedly.

Building the web app was valuable for standardising the routine work and getting it off my plate. But the complex, high-value client work still needs someone who understands both the AI tools and the limitations — and that's the harder skill to replicate.
